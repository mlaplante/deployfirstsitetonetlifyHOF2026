üìÇ Understanding File Paths

File paths tell the browser where to find images, CSS, and other resources. Getting these right is crucial!

‚ùå Absolute Paths (Avoid)

C:/Users/YourName/Desktop/site/img.jpg

file:///Users/you/Documents/photo.png

Won't work when deployed! Only works on your computer.

‚úÖ Relative Paths (Use These)

./images/photo.jpg
(same folder or subfolder)

../assets/logo.png
(go up one folder, then down)

Works everywhere! Moves with your files.

üí° Pro Tip: ./ means "current folder", ../ means "parent folder"

üìö Git & GitHub Quick Start

What's the Difference?

Git = Version Control Tool

Tracks changes to your files over time. Like "undo" on steroids - save snapshots of your project.

GitHub = Cloud Storage

Stores your Git repositories online. Share code, collaborate, and backup your work.

Basic Workflow for This Workshop

# Navigate to your website folder
cd my-website

# Initialize Git (one time only)
git init

# Stage all your files
git add .

# Save snapshot with a message
git commit -m "Initial version of my portfolio"

# Create main branch
git branch -M main

# Connect to GitHub repository (create repo first on github.com)
git remote add origin https://github.com/yourname/portfolio.git

# Upload to GitHub
git push -u origin main

Method 2: GitHub Integration

1 Push to GitHub: Commit your code to a GitHub repository
2 Connect Netlify: Link your GitHub account to Netlify
3 Select Repo: Choose your repository from the list
4 Auto Deploy: Every push to GitHub auto-deploys to Netlify

üîÑ Continuous Deployment: Make changes, push, and your site updates automatically!
